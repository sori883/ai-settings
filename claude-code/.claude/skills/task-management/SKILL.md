# タスク管理スキル

`docs/タスク/` 配下のマークダウンファイルでプロジェクトの実装タスクを管理する。

## タスクファイルの場所

```
docs/タスク/
├── 01_短期記憶【完了】.md
├── 02_中期記憶【完了】.md
├── ...
└── NN_機能名.md          # 未完了
```

## ファイル命名規則

```
{連番}_{機能名}{ステータス}.md
```

| 要素 | 形式 | 例 |
|------|------|------|
| 連番 | 2桁ゼロ埋め | `01`, `08`, `12` |
| 機能名 | 簡潔な日本語 | `短期記憶`, `AWS_CDKインフラ` |
| ステータス | 全タスク完了時のみ `【完了】` を付与 | `【完了】` |

- 未完了タスクが1つでもある場合、ファイル名にステータスは付けない
- 全タスクが完了したら `【完了】` を付ける

## ファイル構成テンプレート

```markdown
# {連番}. {機能名}

{1〜2行の概要説明}

## {連番}.1 セクション名

### {連番}.1.1 タスク名 ✅

- 完了したタスクの要約（箇条書き）

### {連番}.1.2 タスク名

- 未完了タスクの内容（箇条書き）

---

## {連番}.N 検証

- [ ] 検証項目1
- [ ] 検証項目2
- [x] 完了した検証項目
```

### ルール

- 各タスク見出し（`###`）に ✅ を付けて完了を示す
- セクション見出し（`##`）は配下の全タスクが完了したら ✅ を付ける
- 検証セクションはチェックボックス `- [x]` / `- [ ]` を使う
- 詳細なファイルパス・依存関係・成果物は含めてよいが、コスト見積もり・リスク表・ファイル一覧等の補足情報は含めない

## 操作手順

### 1. タスク一覧の確認

```bash
ls docs/タスク/
```

ファイル名からステータスがわかる:
- `【完了】` あり → 全タスク完了
- `【完了】` なし → 未完了タスクあり

### 2. 新規タスクファイルの作成

1. `docs/タスク/` の既存ファイルの連番を確認する
2. 次の連番でファイルを作成する（例: `08_新機能.md`）
3. テンプレートに従って内容を記述する
4. CLAUDE.md の更新は不要（`docs/タスク/` を参照済み）

### 3. タスクの完了マーク

1. 該当タスクの見出しに ✅ を追加する: `### 1.2.3 タスク名` → `### 1.2.3 タスク名 ✅`
2. 検証項目のチェックを更新する: `- [ ]` → `- [x]`
3. セクション内の全タスクが完了したらセクション見出しにも ✅ を追加する
4. ファイル内の全タスクが完了したらファイル名に `【完了】` を追加する

### 4. ファイル名のステータス更新

```bash
# 完了時: ファイル名に【完了】を追加
mv "docs/タスク/08_新機能.md" "docs/タスク/08_新機能【完了】.md"

# 再開時: 【完了】を除去
mv "docs/タスク/08_新機能【完了】.md" "docs/タスク/08_新機能.md"
```
